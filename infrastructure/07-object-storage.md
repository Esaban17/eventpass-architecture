# Object Storage — AWS S3

## 1. Responsibilities

- **Event images** — Store organizer-uploaded event poster images and venue photos. Served to users via Cloudflare CDN with 24h edge cache TTL.
- **QR code tickets** — Store generated QR code images (`tickets/{ticketId}.png`) after purchase confirmation. Referenced in confirmation emails and the buyer's order history page.
- **Frontend static assets** — Store Next.js build artifacts (`_next/static/*`) as the origin for CDN-served JavaScript bundles, CSS, and fonts.
- **Database backups** — Store RDS automated backup snapshots and manual pre-deployment snapshots for disaster recovery.
- **Log archival** — Long-term storage for Loki log chunks and Tempo trace data, providing cost-effective retention beyond the primary observability stack's hot storage.

## 2. Why This Project Needs It

EventPass generates binary artifacts that do not belong in the primary database: event images (uploaded by organizers, ~500KB-2MB each), QR code tickets (generated per purchase, ~5KB each), and static frontend builds (~10MB per deployment). S3 provides virtually unlimited storage at $0.023/GB/month — orders of magnitude cheaper than storing these in PostgreSQL or serving them from ECS task memory.

The QR code storage is particularly important: each ticket's QR code must be accessible from the buyer's order history and the event entry validation scanner. Storing QR codes in S3 with permanent URLs ensures they remain accessible regardless of application deployments or database migrations.

## 3. Technology Choice

| Dimension | AWS S3 | Google Cloud Storage (GCS) | MinIO (self-hosted) |
|-----------|--------|---------------------------|---------------------|
| **Managed/Self-hosted** | Fully managed (AWS) | Fully managed (GCP) | Self-hosted on ECS/EC2 |
| **Operational complexity** | Low — zero infrastructure management, 11 nines durability | Low — similar to S3, different SDK | High — must manage storage nodes, replication, upgrades |
| **Cost at our scale** | ~$5/month (50GB storage + minimal requests) | ~$5/month (comparable pricing) | ~$30/month (ECS task + EBS volumes) |
| **Key differentiating feature** | Native AWS integration (CloudFront, Lambda, RDS backups, lifecycle policies) | Multi-region replication, unified with BigQuery analytics | S3-compatible API, on-premise data sovereignty |

**Decision: AWS S3** — EventPass's infrastructure is entirely on AWS (ECS, RDS, ElastiCache, Amazon MQ). S3 integrates natively with all these services: RDS backups write directly to S3, CloudFront can use S3 as an origin, and ECS tasks access S3 via IAM roles without managing access keys. Using GCS or MinIO would require cross-cloud networking and credential management for no practical benefit.

## 4. Trade-offs

| Advantages | Disadvantages |
|------------|---------------|
| 99.999999999% (11 nines) durability — QR codes and event images are never lost | Eventual consistency for overwrite PUTs — rare but possible stale read after immediate update |
| Lifecycle policies automate cost management — move old logs to Glacier after 90 days | S3 is AWS-specific — migration to another cloud requires tooling (rclone, aws s3 sync) |
| Presigned URLs enable secure, time-limited direct uploads from organizers' browsers | Latency for first-byte retrieval (~50-100ms) — mitigated by Cloudflare edge caching |
| IAM-based access control — ECS tasks access S3 via task role, no credentials in code | No built-in image transformation — organizer-uploaded images must be resized at upload time or via Lambda |
| Cost-effective at scale — storage is $0.023/GB, requests are $0.0004/1K GETs | Request pricing can add up for high-frequency access patterns (mitigated by CDN caching) |

## 5. Integration

- **Ticketing module:** Writes QR code images to `s3://eventpass-tickets/{ticketId}.png` after `OrderConfirmed` event processing. Generates presigned URLs (1h TTL) for buyer access.
- **Event Management module:** Organizers upload event images via presigned URLs generated by the API. Images stored at `s3://eventpass-media/events/{eventId}/poster.jpg`.
- **Next.js frontend:** Static build artifacts deployed to `s3://eventpass-static/_next/static/` and served via Cloudflare CDN (see [02-cdn.md](02-cdn.md)).
- **RDS backups:** Automated daily snapshots and PITR logs stored in `s3://eventpass-backups/rds/`. See [06-primary-database.md](06-primary-database.md).
- **Observability:** Loki log chunks and Tempo trace spans stored in `s3://eventpass-observability/` with lifecycle policy to transition to Glacier after 90 days. See [11-observability-stack.md](11-observability-stack.md).
- **Deployment diagram reference:** See [diagrams/04-deployment.md](../diagrams/04-deployment.md) — S3 sits in the Private VPC zone, accessed by ECS tasks via VPC endpoint.
